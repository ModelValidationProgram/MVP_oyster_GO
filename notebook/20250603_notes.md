### Survival Analysis

#### Calculating Survival

I calculated survival of oysters across time by population, site, and a combination of the two. Survival was calculated at each time point as:

$$Survival_t = \frac{Alive}{Alive + Dead} * Survival_{t-1}$$

These data are exported into multiple formats, specifically "mortality_bags_052925.csv" (has bags, populations, and sites), "mortality_long_bags_052925.csv" (survival by bag, no population or site data attached), and "mortality_long_popsite_05292025.csv" (mean survival by population and site).

When I found a bag with survival higher than 1, that bag was removed from other analysis. This may bias downstream results, as the bag is entirely removed rather than removed from just that time point - I would like to go back to fix this soon, but also will have issues with scaling survival by previous time points that don't exist, so I'm not sure what to do exactly.


#### Modeling Differences in Survival

I used the population level data, rather than the bag level data, to model differential survival, and instead used the pop/site level data (wherein I calculated survival by pooling the number of alive across all bags and the number of dead across all bags, not by taking the average of bags then using those to calculate population level survival). Because of this, I did not need to include bag as a random factor in my model and was able to use regular linear models lm(), followed by a type III ANOVA and Tukey-Kramer tests to look at differential survival across populations, sites, and time. Models took the form: lm(surv ~ bag_site * pop * t, data = surv_mod_ord_rm)

Model selection using Satterwaithe's method showed that all three factors and their two-way (but not three-way) interactions were significant in the final model. Using the same methodology on just the final time point, it was found that the same factors and interactions should be retained in the final model and were significant. Tukey-Kramer models were used to explicitly test for differences between populations at different sites.

### Length Analysis

#### Calculating Length

I calculated the average length of tagged oysters across time in each bag, as well as by population/site. The sample size for these average lengths declined through time, as not all tagged oysters survived, but all bags started with 40 tagged oysters that we were able to track individually through time. I did not calculate growth of individuals through time by looking at individual changes in length, as oyster size is not always well approximated by length * width due to highly variable shell shapes. 

These data are exported as "len_bags_event_2025-06-03.csv" (has bag, monitoring event, and length), "lengths_long_2025-06-04.csv" (expanded dataset linking individual tags to their respective bags, populations, lengths, widths, and monitoring events), and "len_sites_2025-06-03.csv" (site, pop, length, monitoring event).

#### Modeling Differences in Length

I did use bag level data to model length, so I did need to use linear mixed models lmer() from the package cars. The model took the form: lmer(length ~ bag_site * pop * t + (1|bags_label), data = lengths_mod_narm_nodead)

I used a type III ANOVA to look at significance of variables, finding that all variables and their interactions were significant.  Using the same methodology on just the final time point, I again found all variables and interactions were significant. Tukey-Kramer models were used to explicitly test for differences between populations at different sites.

### LEA GO

LEAoffset is based on estimates of environmental effect sizes obtained from a latent-factor-mixed-model (LFMM), and the relationship inferred by the LFMM is then used to fit and predict allelic variation at all loci (Gain & Francois 2021). This method calculates offsets at the individual level, which are then averaged to calculate a population level offset. LEA requires .lfmm format genomic data and .env format environmental data. I used the lfmm2() function to train the lfmm models, then the genetic.gap() function to calculate genomic offsets based on these trained models. I specifically calculated genomic offset between experimental populations' home sites and both Lewisetta and the York River (separately).

### GF GO

GFoffset uses regression trees to fit a model of association between individual response variables (loci) to a multivariate set of predictors (climate variables) which can then be used to create a prediction of allelic turnover as a proxy of future population maladaptation, or genomic offset (Fitzpatrick & Keller 2015, Capblancq et. al. 2020). GradientForests can be built on either population level (allele frequency) or individual level (genotype) genomic data and environmental data. I first calculated allele frequencies with a custom function, then used gradientForest() to train separate models using AF and GENO data. I then used the predict() function to look at allelic turnover for populations/individuals in their home environments (GO.h) and in the two common garden experimental sites (GO.l, GO.y). Genomic offset was then calculated as the euclidean distance between GO.h and either GO.l or GO.y. I associated the appropriate population and site metadata with the calculated GOs. These results are exported as "offsetsMVP_2025-06-03.csv"

The gradientForest() training is a computationally demanding function, so I can only run the genotype model on the cluster.
